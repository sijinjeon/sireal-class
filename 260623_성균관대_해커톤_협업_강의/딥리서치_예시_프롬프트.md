# 딥리서치 예시 프롬프트

> 아이데이션에서 나온 아이디어를 심사 기준에 대입해 검증할 때 사용한다.
> 아이디어가 나올 때마다 즉시 돌린다.

---

## 사용법

1. 아래 프롬프트에서 `[아이디어]` 부분을 실제 아이디어로 교체한다.
2. Gemini Deep Research / ChatGPT Deep Research / Claude / Perplexity 중 하나에 입력한다.
3. 결과를 노션에 저장하고 심사 기준표에 대입한다.

---

## 1. 종합 검증 (한 번에 다 보고 싶을 때)

```
[아이디어]에 대해 아래 5가지 기준으로 딥리서치를 진행해 줘.

1. 문제 정의 및 사회적 필요성: 이 문제가 실제로 존재하는가? 얼마나 많은 사람이 겪고 있는가?
2. 창의성 및 독창성: 유사한 서비스나 솔루션이 이미 존재하는가? 차별점은 무엇인가?
3. AI 활용의 적절성 및 기술 타당성: AI를 어떻게 활용할 수 있으며, 기술적으로 구현 가능한가?
4. 실현 가능성 및 완성도: 해커톤(24~48시간) 내에 MVP를 만들 수 있는가?
5. 기대 효과 및 파급력: 이 솔루션이 실제로 적용되면 어떤 변화가 생기는가?

각 항목별로 근거 자료와 함께 서술하고, 마지막에 종합 의견을 한 단락으로 정리해 줘.
```

---

## 2. 기준별 단독 프롬프트

### 2-1. 문제 정의 및 사회적 필요성

```
[아이디어]가 해결하려는 문제에 대해 리서치해 줘.

- 이 문제를 겪는 사람이 실제로 얼마나 되는가? (통계, 설문, 뉴스 등 근거 포함)
- 기존에 이 문제를 해결하려는 시도가 있었는가?
- 왜 아직 해결되지 않았는가?
```

### 2-2. 창의성 및 독창성

```
[아이디어]와 유사한 서비스나 제품을 조사해 줘.

- 국내외 유사 사례를 최대한 나열해 줘.
- 각 사례의 한계나 단점은 무엇인가?
- [아이디어]가 기존 사례와 다른 점은 무엇이 될 수 있는가?
```

### 2-3. AI 활용의 적절성 및 기술 타당성

```
[아이디어]를 구현할 때 AI를 어떻게 활용할 수 있는지 분석해 줘.

- 어떤 AI 기술(LLM, 이미지 생성, 음성 인식 등)이 적합한가?
- 현재 공개된 API나 오픈소스로 구현 가능한가?
- 기술적으로 가장 어려운 부분은 무엇인가?
```

### 2-4. 실현 가능성 및 완성도

```
[아이디어]의 핵심 기능 1~2개를 24~48시간 해커톤 안에 구현한다고 가정할 때:

- MVP(최소 기능 제품)로 줄이면 무엇을 만들어야 하는가?
- 어떤 기술 스택이 가장 빠르게 구현 가능한가?
- 팀이 비개발자를 포함한 혼합 팀이라면 어떻게 역할을 나눌 수 있는가?
```

### 2-5. 기대 효과 및 파급력

```
[아이디어]가 실제로 출시된다고 가정하고 기대 효과를 분석해 줘.

- 직접적인 수혜자는 누구이며, 삶이 어떻게 달라지는가?
- 사회적·경제적 파급 효과는 무엇인가?
- 심사위원이나 투자자 입장에서 이 아이디어의 매력 포인트는 무엇인가?
```

---

## 3. 경쟁사·시장 분석 (선택)

```
[아이디어] 관련 시장을 분석해 줘.

- 시장 규모와 성장률 (가능하면 수치 포함)
- 주요 플레이어와 각각의 강점·약점
- 이 시장에서 스타트업이나 해커톤 팀이 비집고 들어갈 수 있는 틈새는 어디인가?
```

---

## 4. 발표용 한 줄 요약 생성 (딥리서치 결과 후속 활용)

딥리서치 결과가 나온 뒤, 아래 프롬프트로 발표 문구를 뽑는다.

```
위 리서치 결과를 바탕으로 다음을 작성해 줘.

1. 아이디어를 한 문장으로 설명하는 엘리베이터 피치 (30초 분량)
2. 심사위원에게 어필할 수 있는 핵심 강점 3가지 (불릿 포인트)
3. 예상 반론 2가지와 그에 대한 답변
```
